1. Overzicht

De Microsoft 365 Calendar Integration app voor Nextcloud maakt het mogelijk om Microsoft 365 gebruikers te zoeken en toe te voegen als deelnemers aan agenda-items in de Nextcloud Calendar. Daarnaast wordt de beschikbaarheid (free/busy) van MS365-gebruikers direct zichtbaar in de Nextcloud Calendar.

Functionaliteiten

Functie Omschrijving
Deelnemers zoeken Zoek Microsoft 365 gebruikers bij het toevoegen van deelnemers aan een agenda-item
Free/Busy Bekijk de beschikbaarheid van MS365-gebruikers direct in de Nextcloud Calendar
OAuth2 met PKCE Veilige per-gebruiker authenticatie met Microsoft Entra ID
CalDAV-compatibel Werkt transparant met Apple Calendar, Thunderbird en andere CalDAV-clients

Let op: Elke Nextcloud-gebruiker koppelt individueel zijn/haar eigen Microsoft 365-account. Er is geen centrale service-account nodig.

3. Microsoft Entra ID configuratie

Onderstaande stappen worden uitgevoerd door een Microsoft 365 beheerder met toegang tot de Azure Portal.

3.1 App Registration aanmaken

  1. Ga naar de Azure Portal
  2. Navigeer naar Microsoft Entra ID → App registrations → New registration
  3. Vul de volgende gegevens in:
Veld Waarde
Name Nextcloud Calendar Integration (of een beschrijvende naam naar keuze)
Supported account types Accounts in this organizational directory only (Single tenant)
Redirect URI Platform: Web
URI: https://<uw-nextcloud-domein>/apps/nc_ms365_calendar/oauth/callback

Belangrijk: De Redirect URI moet exact overeenkomen met het domein van uw Nextcloud-installatie. Vervang <uw-nextcloud-domein> door het werkelijke domein, bijvoorbeeld: https://cloud.organisatie.nl/apps/nc_ms365_calendar/oauth/callback

  1. Klik op Register

3.2 API-machtigingen instellen

Na het aanmaken van de App Registration moeten de juiste API-machtigingen worden toegevoegd:

  1. Ga naar uw App Registration → API permissions → Add a permission
  2. Kies Microsoft Graph → Delegated permissions
  3. Voeg de volgende machtigingen toe:
Machtiging Doel Admin Consent
User.ReadBasic.All Gebruikers zoeken in de directory Vereist
People.Read Relevante contacten ophalen Nee
Calendars.Read Free/busy informatie opvragen Nee
Calendars.Read.Shared Free/busy voor gedeelde agenda's Nee
offline_access Refresh tokens voor langdurige sessies Nee
  1. Klik op Grant admin consent for [uw organisatie]
  2. Bevestig met Yes

Let op: De machtiging User.ReadBasic.All vereist admin consent. Zonder deze goedkeuring kunnen gebruikers geen MS365-collega's zoeken als deelnemers.

Toelichting op de machtigingen: Alle machtigingen zijn van het type Delegated, wat betekent dat ze werken namens de ingelogde gebruiker. De app kan nooit meer doen dan waartoe de gebruiker zelf gemachtigd is. Er worden geen Application-machtigingen gebruikt.

3.3 Client Secret aanmaken

  1. Ga naar uw App Registration → Certificates & secrets
  2. Klik op New client secret
  3. Voer een beschrijving in (bijv. Nextcloud Calendar) en kies een verlooptermijn
  4. Kopieer de secret value onmiddellijk — deze wordt slechts eenmaal weergegeven

Waarschuwing: De Client Secret wordt na het aanmaken slechts eenmaal getoond. Sla deze veilig op. Als u de waarde kwijtraakt, moet u een nieuwe secret aanmaken.

Aanbeveling: Stel een herinnering in voor de verlooptermijn van het secret. Na verlopen moet een nieuw secret worden aangemaakt en in Nextcloud worden bijgewerkt. Een verlopen secret zorgt ervoor dat gebruikers hun MS365-account niet meer kunnen koppelen.

3.4 Tenant ID en Client ID noteren

Ga naar uw App Registration → Overview en noteer de volgende waarden:

Veld Waar te vinden Voorbeeld
Tenant ID Overview → Directory (tenant) ID a1b2c3d4-e5f6-7890-abcd-ef1234567890
Client ID Overview → Application (client) ID 12345678-abcd-efgh-ijkl-123456789012
Client Secret Certificates & secrets (stap 3.3) uw-gekopieerde-secret-waarde

Klaar! U heeft nu alle gegevens die nodig zijn voor de Nextcloud-configuratie: Tenant ID, Client ID en Client Secret.

4. Nextcloud configuratie

4.1 App installeren

De app wordt geinstalleerd door de Nextcloud-beheerder:

  1. Kloon de repository in de Nextcloud apps/ directory:

    cd /var/www/nextcloud/apps
    git clone https://gitea.rikdekker.nl/sam/nc_ms365_calendar.git

  2. Installeer dependencies en bouw de frontend:

    cd nc_ms365_calendar
    composer install --no-dev
    npm install
    npm run build

  3. Activeer de app:

    occ app:enable nc_ms365_calendar

4.2 Beheerdersinstellingen

Ga naar Beheer → Microsoft 365 Calendar Integration en vul de volgende velden in:

Veld Waarde Toelichting
Tenant ID Uw Microsoft Entra ID Tenant ID Zie stap 3.4
Client ID Het Application (client) ID van uw App Registration Zie stap 3.4
Client Secret Het client secret van uw App Registration Zie stap 3.3
Cache TTL 300 (standaard) Hoe lang zoekresultaten gecachet worden in seconden (60–3600)

Klik op Save om de instellingen op te slaan.

Redirect URI: Op de instellingenpagina wordt automatisch de juiste Redirect URI getoond. Controleer of deze overeenkomt met wat u in stap 3.1 heeft ingesteld in de Azure Portal.

4.3 Gebruikerskoppeling

Elke gebruiker koppelt zelf zijn/haar Microsoft 365-account:

  1. Ga naar Persoonlijke instellingen → Microsoft 365 Calendar
  2. Klik op Connect to Microsoft 365
  3. Log in met uw Microsoft-account en verleen de gevraagde machtigingen
  4. Na succesvolle koppeling wordt de status weergegeven

Let op: Gebruikers moeten individueel inloggen met hun eigen Microsoft 365-account. De app slaat per gebruiker een versleuteld OAuth-token op.

5. Werking

5.1 Deelnemers zoeken

Wanneer een gebruiker deelnemers zoekt in de Nextcloud Calendar, worden automatisch ook Microsoft 365 gebruikers doorzocht:

  1. De app doorzoekt de People API voor vaak gecontacteerde personen
  2. Parallel wordt de Users API doorzocht op weergavenaam en e-mailadres
  3. Resultaten worden samengevoegd, gedupliceerden verwijderd en gecachet (standaard 5 minuten)

Technisch: De app registreert een virtueel adresboek via de Nextcloud Contacts Manager. Dit adresboek wordt automatisch doorzocht wanneer de Calendar-app om contacten vraagt.

5.2 Beschikbaarheid (Free/Busy)

De beschikbaarheid van MS365-gebruikers wordt automatisch weergegeven in de Nextcloud Calendar:

  1. De app registreert een Sabre DAV plugin die CalDAV free/busy-verzoeken onderschept
  2. Externe deelnemers (niet in Nextcloud) worden geidentificeerd
  3. De Microsoft Graph API (/me/calendar/getSchedule) wordt bevraagd
  4. MS365-beschikbaarheid wordt samengevoegd met de Nextcloud-response

Statusmapping

MS365 Status CalDAV Status Weergave
busy BUSY Bezet
tentative BUSY-TENTATIVE Onder voorbehoud
oof (Out of Office) BUSY-UNAVAILABLE Niet beschikbaar
workingElsewhere BUSY-TENTATIVE Werkt elders

CalDAV-compatibel: Omdat de free/busy-informatie via het standaard CalDAV-protocol wordt aangeboden, werkt dit transparant met alle CalDAV-clients: Apple Calendar, Thunderbird, en andere.

6. Technische architectuur

Dit hoofdstuk beschrijft de technische werking van de app voor beheerders die willen begrijpen hoe data stroomt tussen Nextcloud, de gebruiker en Microsoft 365.

6.1 OAuth2-authenticatiestroom

De app gebruikt OAuth 2.0 Authorization Code Flow met PKCE (Proof Key for Code Exchange) voor authenticatie met Microsoft Entra ID. PKCE is een uitbreiding op het standaard OAuth-protocol die beschermt tegen onderschepping van de autorisatiecode.

Stroom

  1. Gebruiker klikt op Connect to Microsoft 365 in persoonlijke instellingen
  2. De app genereert een cryptografisch willekeurige code verifier (64 bytes) en een state parameter (32 bytes)
  3. Een code challenge (SHA-256 hash van de verifier) wordt meegestuurd naar Microsoft
  4. Gebruiker logt in bij Microsoft en verleent toestemming
  5. Microsoft stuurt een autorisatiecode terug naar de Nextcloud callback-URL
  6. De app valideert de state parameter (CSRF-bescherming) en wisselt de code + verifier in voor tokens
  7. Tokens worden versleuteld opgeslagen (zie sectie 7.1)

Waarom PKCE? Bij standaard OAuth kan een kwaadwillende app op hetzelfde apparaat de autorisatiecode onderscheppen. Met PKCE kan alleen de partij die de oorspronkelijke code verifier bezit de code inwisselen voor tokens. De verifier wordt nooit via het netwerk verstuurd — alleen de SHA-256 hash.

Token-levenscyclus

Token Levensduur Vernieuwing
Access token Circa 1 uur (bepaald door Microsoft) Automatisch vernieuwd 5 minuten voor verlopen
Refresh token 90 dagen (standaard Microsoft-beleid) Vernieuwd bij elk gebruik; verlopen vereist hernieuwde koppeling

Let op: Als een refresh token verloopt of wordt ingetrokken (bijv. door een Microsoft-beheerder), moet de gebruiker opnieuw koppelen via Persoonlijke instellingen. De app detecteert dit automatisch en toont de juiste status.

6.2 Microsoft Graph API-communicatie

Alle communicatie met Microsoft 365 verloopt via de Microsoft Graph REST API (https://graph.microsoft.com/v1.0). De app maakt gebruik van de volgende endpoints:

Endpoint Methode Doel
/me/people GET Zoeken naar vaak gecontacteerde personen
/users GET Zoeken naar gebruikers op naam of e-mailadres
/users/{id}/photo/$value GET Profielfoto ophalen
/me/calendar/getSchedule POST Beschikbaarheid opvragen (free/busy)

Foutafhandeling en herhaalpogingen

De app implementeert robuuste foutafhandeling voor API-communicatie:

HTTP-status Reactie
401 Unauthorized Token automatisch vernieuwd, verzoek opnieuw geprobeerd (max 1x)
429 Too Many Requests Wacht op Retry-After header, exponential backoff
503/504 Service Unavailable Exponential backoff: 1s, 2s, 4s (max 3 pogingen)
403 Forbidden Geen herhaalpogingen; machtiging ontbreekt (zie sectie 3.2)

6.3 Caching

De app cachet resultaten om het aantal API-aanroepen te beperken en de prestaties te verbeteren:

Data TTL Toelichting
Zoekresultaten 5 minuten (instelbaar via Cache TTL) Per gebruiker, per zoekterm
Free/busy 2 minuten Per gebruiker, per e-mailadres, per tijdvenster
Profielfoto's 24 uur Per MS365-gebruiker

Aanbeveling: Voor Nextcloud-installaties met meerdere servers wordt een distributed cache aanbevolen (Redis of Memcached). Zonder distributed cache worden zoekresultaten per server gecachet, wat leidt tot meer API-aanroepen.

7. Beveiliging

Dit hoofdstuk beschrijft de beveiligingsmaatregelen die de app implementeert om gebruikersgegevens en tokens te beschermen.

7.1 Token-opslag en versleuteling

Aspect Implementatie
Access token Versleuteld opgeslagen met Nextcloud ICrypto
Refresh token Versleuteld opgeslagen met Nextcloud ICrypto
Client Secret Opgeslagen als sensitive app-configuratie (versleuteld door Nextcloud)
PKCE-verifier Alleen in server-side sessie; wordt na gebruik direct verwijderd
State parameter Alleen in server-side sessie; wordt na validatie direct verwijderd

Let op: De versleuteling is afhankelijk van de Nextcloud secret in config.php. Zorg ervoor dat deze sleutel veilig bewaard wordt en onderdeel is van uw backup-strategie. Als deze sleutel verloren gaat, moeten alle gebruikers opnieuw koppelen.

Bij het ophalen van instellingen via de admin-interface wordt het Client Secret gemaskeerd weergegeven (********). Het opslaan van een gemaskeerde waarde wordt automatisch genegeerd, zodat het bestaande secret niet overschreven wordt.

7.2 Least privilege

De app is ontworpen volgens het principe van minimale rechten:

Principe Implementatie
Delegated permissions Alle API-machtigingen zijn delegated, niet application. De app kan nooit meer dan waartoe de gebruiker zelf gemachtigd is
Alleen lezen De app vraagt uitsluitend leesmachtigingen op (Read). Er worden geen agenda-items, contacten of gebruikersgegevens gewijzigd in Microsoft 365
Single tenant De App Registration is beperkt tot uw eigen organisatie. Gebruikers van andere tenants kunnen de app niet gebruiken
Per-gebruiker tokens Elke gebruiker heeft eigen tokens. Er is geen gedeelde service-account met brede toegang
Read-only adresboek Het MS365-adresboek in Nextcloud is read-only. Er kunnen geen contacten worden aangemaakt, gewijzigd of verwijderd

7.3 Netwerkbeveiliging

Aspect Implementatie
Transport Alle communicatie met Microsoft via HTTPS/TLS (afgedwongen door Microsoft Graph API)
CSRF-bescherming OAuth state parameter (32 bytes, cryptografisch willekeurig) voorkomt cross-site request forgery
XXE-preventie XML-parsing gebruikt LIBXML_NONET | LIBXML_NOENT vlaggen om externe entiteiten en netwerktoegang te blokkeren
Uitgaand verkeer De server maakt alleen verbinding met login.microsoftonline.com en graph.microsoft.com (poort 443)

Firewall-configuratie: Als uw Nextcloud-server achter een firewall staat, zorg dan dat uitgaand HTTPS-verkeer (poort 443) is toegestaan naar:

  • login.microsoftonline.com — OAuth-authenticatie
  • graph.microsoft.com — Microsoft Graph API

Er is geen inkomend verkeer van Microsoft nodig.

7.4 Foutafhandeling en logging

Aspect Implementatie
Gevoelige data in logs Tokens, secrets en gebruikersgegevens worden nooit gelogd. Alleen endpoints, HTTP-statuscodes en foutcategorieën worden vastgelegd
Foutmeldingen naar gebruiker Gebruikers zien generieke foutmeldingen; technische details staan alleen in de serverlog
Token-intrekking Bij een HTTP 401 tijdens token-refresh wordt de gebruiker automatisch ontkoppeld en moeten alle tokens opnieuw worden aangemaakt
Mislukte decryptie Als token-decryptie mislukt (bijv. na sleutelwijziging) wordt een leeg resultaat geretourneerd en de fout gelogd, zonder crash

Samenvatting: De app slaat geen wachtwoorden op, gebruikt alleen versleutelde tokens, communiceert uitsluitend via HTTPS, en kan geen gegevens wijzigen in Microsoft 365. Elke gebruiker autoriseert individueel en kan de koppeling op elk moment verbreken.

8. Probleemoplossing

Probleem Mogelijke oorzaak Oplossing
Gebruikers kunnen niet inloggen bij MS365 Onjuiste Redirect URI Controleer of de Redirect URI in Azure exact overeenkomt met die op de Nextcloud-instellingenpagina
Foutmelding "AADSTS7000215" Ongeldig Client Secret Maak een nieuw Client Secret aan in Azure en werk deze bij in de Nextcloud-instellingen
Geen zoekresultaten voor MS365-gebruikers Admin consent niet verleend Ga naar Azure → API permissions en klik op "Grant admin consent"
Free/busy werkt niet Gebruiker niet gekoppeld Controleer of de gebruiker zijn MS365-account heeft gekoppeld via Persoonlijke instellingen
Free/busy werkt niet Calendars.Read machtiging ontbreekt Voeg de machtiging toe in Azure en verleen admin consent
Token verlopen Refresh token ongeldig Gebruiker moet opnieuw koppelen via Persoonlijke instellingen → Disconnect en opnieuw Connect
App niet zichtbaar in Nextcloud App niet geactiveerd Voer occ app:enable nc_ms365_calendar uit op de server
Netwerk timeout bij MS365-verzoeken Firewall blokkeert uitgaand verkeer Zorg dat de server graph.microsoft.com en login.microsoftonline.com kan bereiken (HTTPS, poort 443)

Logbestanden: Controleer de Nextcloud-logbestanden voor gedetailleerde foutmeldingen: /var/www/nextcloud/data/nextcloud.log of via Beheer → Logging.
